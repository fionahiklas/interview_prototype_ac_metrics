 AC Device : Actor
 Register Device : Use Case

The device API expects a serial number and registration token from the AC unit.  The validity of these are checked and, on success, an API token is returned which allows the AC unit to make further calls to the API.

 Authenticate Device : Use Case

The AC unit's HTTP client must send the API token it received on registration in order to obtain a valid token that can be used to send periodic data.  

 

This second token will have a limited life span and will need to be refreshed periodically by calling the authenticate API again to obtain a new token.

 

The validity period for the data-sending token needs to be extractable by the AC unit HTTP client so that it can obtain a new value in plenty of time.

 Send Periodic Data : Use Case

The AC unit will make a HTTP call with the token it obtained from authenticating.

 Send bulk data : Use Case

Bulk data is simply accumulated from when the AC unit was unable to connect to the device API.  Rather than 1 item in the call there could be up to 500 (approx 8hours worth of data).
