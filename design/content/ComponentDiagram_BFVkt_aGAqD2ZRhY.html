  RRD Files : Component
  Node JS Frontend : Component

Node.js application using the following tech stack

 

- express for HTTP support including authentication

- socket.io for websockets support

- rrdtool and standard file IO for reading sensor data

- redis for pubsub/alerts

 

Session data must be encoded in the cookies, no session state will be held in the application or redis

  Ruby Device API : Component

Ruby application using the following

 

- eventmachine for the web server

- thin or similar web server

- grape for REST API

- grape-swagger for API documentation

- rrd-ffi for writing sensor data to files

Stores data from devices in round-robin archives to allow results to be fetched and graph data to be generated in the front end

  Redis Store : Component
  N/A : Note

Can store device tokens for login as well as providing a service for pub/sub communication between the device API service and front end to send notification of alerts which can then be retrieved.  Alerts can be stored here per device also.  Need also a list of alerting devices here too

  Notifications : Port
  N/A : Note

Use websockets for real time alerts.  When scaling the notifications will rely on the alerts being sent to all nodes using pub/sub.

 

The client websocket connection also needs some form of keep-alive message so that fail-over to other nodes can be carried out by reconnecting if the current host goes down.

 

The websocket connection does pin the client to a given node though
